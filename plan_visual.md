# Gistify Visual Redesign Audit — Plan

## Goal
Transform Gistify (gistify.pro) from a dark-theme fintech dashboard into a visually **classic, refined, premium** experience. Produce a comprehensive visual redesign brief covering color, typography, spacing, components, animations, and responsive behavior.

## Stage 1 — Parallel Visual Exploration (4 agents)
Each agent navigates gistify.pro via WebBridge, takes screenshots, and writes a focused visual analysis.

- **Agent_Visual_Landing**: Landing page (hero, value props, CTA, footer). Compare to modern classic fintech references (Stripe, Linear, Notion, Pitch). Analyze color palette, typography scale, animation quality, CTA hierarchy.
- **Agent_Visual_Dashboard**: App dashboard (nav bar, tabs, cards, data tables, badges). Analyze component consistency, spacing rhythm, glassmorphism vs flat design, data visualization density, information hierarchy.
- **Agent_Visual_Responsive**: Mobile/tablet responsive behavior. Analyze layout breakpoints, font scaling, touch targets, card stacking, nav collapse, horizontal scroll issues.
- **Agent_Visual_Details**: Micro-details — color contrast, hover/focus states, loading skeletons, empty states, iconography, border radius consistency, shadow depth, transition smoothness, animation timing.

## Stage 2 — Synthesis
Merge all agent findings into a unified visual redesign brief.

## Stage 3 — Report Writing
Write a comprehensive Markdown report (GISTIFY_VISUAL_REDESIGN_BRIEF.md) with:
- Executive summary
- Current state visual analysis (with screenshots)
- Redesign strategy (classic, refined, premium)
- Color palette recommendation (with hex codes)
- Typography system (font stack, scale, weights)
- Spacing & layout system (grid, rhythm, density)
- Component redesign specs (cards, buttons, nav, tables, badges)
- Animation & micro-interaction specs
- Responsive behavior specs
- Before/after visual recommendations
- Implementation priority (P0-P3)

## Output
GISTIFY_VISUAL_REDESIGN_BRIEF.md saved to workspace.
