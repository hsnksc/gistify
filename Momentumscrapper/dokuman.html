1. Proje Özeti

NASDAQ Momentum Scanner, NASDAQ borsasında günün ilk yarım saatinde (09:30–10:00 EST) en güçlü momentumu gösteren hisseleri bulan, çift yönlü (CALL/PUT) opsiyon stratejisi üreten, AI katalizör ve mikro yapı analizi ile zenginleştirilmiş web tabanlı bir analiz platformudur.

3. Algoritma Mimarisi

5 Katmanlı Yapı

Katman 1: Veri Çekme (dataProviders.ts)

Yahoo Finance → Massive → TwelveData → Alphavantage (fallback zinciri). CORS Proxy: AllOrigins → corsproxy.io

Katman 2: Analiz (momentum.ts)

11 Faktör skorlama + Bear skoru (v4.2) + Persistence skoru (v4.2) + AI Catalyst (v4.3) + Microstructure (v4.3) + RSI RED filtresi (v4.1) = Raw Score

Katman 3: Doğrulama (sanityGate.ts)

NaN/Infinity kontrolü, aralık clamp [0, 100], ağırlık normalize

Katman 4: Skorlama (scoreConfig.ts)

Confidence Score (4 bileşen) + Ranking Score (4 bileşen) + Sinyal belirleme (9 sinyal)

Katman 5: Çıktı (optionsStrategies.ts)

CALL setup + PUT setup + POP hesaplama + Kelly sizing + Execution plan

Veri Akışı Özeti

TARA butonu → Paralel tarama (30 hisse) → Her hisse için 4 API denenir → 11 faktör skorlanır + Bear + Persistence + AI Catalyst + Microstructure + RSI RED → Sanity Gate → Confidence + Ranking → CALL/PUT setup üretimi → Sonuç tablosu

5. Versiyon Geçmişi

5.1 v4.1 — RSI RED Filtresi

if (rsi14 >= 80) { signal = "OVERBOUGHT_RED"; score = 0; }
else if (rsi14 >= 75) { signal = "CAUTION_HOT"; score = raw * 0.3; }
else if (rsi14 <= 25) { signal = "OVERSOLD_CAUTION"; score = raw * 0.5; }
Amaç

RSI 80+ hisselere "gir" demek yerine otomatik engel koymak. Kullanıcı eleştirisi: "RSI 90'a gir demek saçmalık" → çözüm: skor 0.

8. Opsiyon Stratejileri

Strateji Seçim Mantığı

DurumCALLPUT
Skor ≥ 75 + RVOL ≥ 2 + RSI 50–80Long Call—
Skor ≥ 55 + VWAP > 0Bull Call Spread—
Bear ≥ 75 + RVOL ≥ 2 + RSI ≤ 50—Long Put
Bear ≥ 55 + VWAP < 0—Bear Put Spread
Her ikisi ortaBull Put Spread (konservatif)

POP Hesaplama

POP = P(fiyat > breakeven at expiration)
ITM call: ~65% POP
OTM call: distance / (dailyMove × sqrt(DTE/5))

Kelly Sizing

winProb = POP / 100
edge = winProb - (1 / (R/R + 1))
kelly = edge / (1 / R/R) × 100
KellyPozisyon
≥ 2.0%NLV %2.0
≥ 1.5%NLV %1.5
≥ 1.0%NLV %1.0
≥ 0.5%NLV %0.5
< 0.5%NLV %0.3 (küçük)

Vade Kuralları

< 7 gün + RSI 80+AÇMA (çift risk)
< 14 günUyarı (theta yaklaşıyor)

10. Sanity Gate

Dosya: src/lib/sanityGate.ts (231 satır)

Güvenlik Fonksiyonları

isSafeNumber(v)     → number ve NaN/Infinity değilse true
safeNumber(v, fb)   → Güvensizse fallback değer döndür
clamp100(v)         → [0, 100] aralığına zorla

Kontrol Sırası

  1. Her faktör skoru kontrol et (NaN/Infinity/null → 50)
  2. Aralık dışıysa clamp ([0,100] içine zorla)
  3. Ağırlıkları kontrol et (toplam 1.00 değilse normalize)
  4. Final skoru clamp et
  5. Yahoo veri yapısı kontrolü (JSON, meta, null oranı <%30)
Örnek: NaN Kontrolü

Eğer RVOL hesaplaması sonucu NaN çıkarsa (örneğin hacim verisi eksik), sistem otomatik olarak 50 puan atar (nötr). Bu sayede tek bir faktörün hatası tüm skoru çökertmez.
